## What was built (2026-06-23)
- Next.js 15 (App Router) + React 19 + Tailwind v4
- 5-question quiz → 45 unique paths (5 tracks × 3 levels × 3 depths)
- 52 hand-curated YouTube videos across 6 topics
- Shareable URL state (`?p=XXX` encodes path in 3 chars)
- Light + dark mode (auto via `prefers-color-scheme`)
- WCAG AA accessibility (skip link, focus rings, 44×44px touch targets, reduced motion)
- YouTube embeds via `youtube-nocookie` (privacy)

## Architectural decisions
- **Server component reads `?p=XXX`** at request time, so shared URLs render correctly on first paint (no flash)
- **Dedup in Results.tsx** puts each video in the section matching its primary topic (first entry in the video's own `topics[]` array), not TOPICS order
- **`filterByTopic` in paths.ts** sorts by primary-topic-first, then fewer-topics, to avoid videos tagged with work+philosophy from being picked up as pure philosophy

## Bugs found and fixed in this session
1. **`TRACK_CHARS` typo** — was `"pcwbic"` (6 chars) but `TRACK_INDEX` had 5 tracks. So `work` (index 1) encoded as `c` (curious). Fixed to `"pwbic"`. Regression test: `REGRESSION: encoding 'work' track produces 'w'`.
2. **Philosophy section disappearing** — `work-newcomer-2` has `topics: ["work", "philosophy"]` and was being picked up by the philosophy filter before pure-philosophy videos. Fixed by sorting candidates by primary-topic-first.
3. **Dedup placing videos in wrong topic section** — original logic used "first topic in TOPICS order" but should be "first topic in the video's own topics[]" (the video author's primary tag).

## Patterns to keep applying
1. **Run the regression test on the next session.** `npm run test` exercises all 45 paths.
2. **Add a regression test for any bug you find**, with a "REGRESSION:" prefix in the test name, so it can't silently come back.
3. **Verify external resources before claiming they work.** I made up YouTube IDs that pass shape validation but don't resolve. Always click through to confirm.
4. **Type-asserting object keys to value types is a lie.** `Object.keys(X) as Track[]` does not convert strings to the value type. Use `Object.values(X)` when you want the values.
